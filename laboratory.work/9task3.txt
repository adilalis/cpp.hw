1)По значению vs const ссылка
По значению — копия (медленно, не меняет оригинал).
const & — без копии (быстро, только чтение).
2)Почему const std::string&?
Чтобы не копировать строку и не изменять её.
3)Если передать по значению (3.1)?
Изменится только копия, оригинал останется прежним.
4)Почему нельзя возвращать указатель на локальную переменную?
Она удаляется после функции → указатель становится недействительным.
5)Опасность substr() в циклах
Создаёт новые строки → медленно и много памяти.
